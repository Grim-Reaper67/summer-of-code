# summer-of-code

# DSA Learning and Practice

## Overview

This repository contains the work completed so far as part of my Data Structures and Algorithms practice. I solved problems on LeetCode across a variety of topics and wrote the solutions in Python.

The main focus was not only on getting the correct output, but also on understanding the basic idea behind each problem, comparing different possible approaches, and becoming more aware of time and space complexity.

## Topics Practised

The problems covered a range of topics, including:

* Arrays and strings
* Frequency counting
* Prefix sums
* Sliding windows and two pointers
* Sorting
* Binary search
* Stacks, queues, and deques
* Recursion and backtracking
* Binary trees and Binary Search Trees
* Greedy algorithms
* Heaps
* Basic Dynamic Programming

For Dynamic Programming, I completed a few introductory problems such as:

* Climbing Stairs
* Min Cost Climbing Stairs
* House Robber

These problems helped me understand the basic ideas of states, recurrence relations, base cases, memoisation, and tabulation.

## Problem Solving Progress

I was able to solve many easy-level questions and became more comfortable with basic array traversal, loops, dictionaries, sets, sorting, binary search, and simple recursion.

Medium level questions were more challenging, especially when the required pattern was not immediately clear. Problems involving sliding windows, monotonic stacks, binary search on the answer, backtracking, and combinations of multiple techniques usually required more time and explanation.

When I got stuck, I used ChatGPT mainly to understand the starting intuition, identify the relevant technique, or break the problem into smaller parts. After understanding the basic direction, I tried to rewrite the solution in my own style, trace the code with examples, and understand the complexity rather than simply copying the answer.

## Main Learning

The biggest learning from this work was that getting the correct answer is not enough. A solution can still fail because of Time Limit Exceeded, unnecessary memory usage, repeated calculations, invalid indexing, incorrect base cases, excessive recursion depth, or a poor choice of data structure.

I also learned that there can be multiple correct solutions to the same problem. A brute force solution is often useful for understanding the question, but it may later need to be improved using sets, prefix information, sliding windows, binary search, stacks, heaps, or another technique.

The next goal is to practise more medium-level questions, depend less on hints, and improve my ability to identify the correct pattern independently.
